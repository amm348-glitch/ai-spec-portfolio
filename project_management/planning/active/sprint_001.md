# Sprint 001 — Scaffold Site + Initial Deployment

## Goal
Create the initial site scaffold, wire up Pico.css, and deploy a live version using GitHub Pages.

## Scope
### In Scope
- Create `index.html`
- Add Pico.css via CDN
- Create all required page sections (empty or placeholder content allowed)
- Push to GitHub
- Enable GitHub Pages

### Out of Scope
- Final copywriting
- Case study content
- Visual polish
- Screenshots or media assets

## Tasks
1. Create `index.html` in repo root
2. Add HTML boilerplate and Pico.css CDN
3. Add semantic sections matching the spec:
   - Header / Metadata
   - Methodology
   - Featured Project
   - Toolbox / Stack
   - Footer
4. Commit changes
5. Enable GitHub Pages
6. Verify live URL loads

## Files to Touch
- `index.html`
- Repository settings (GitHub Pages)

## Acceptance Criteria
- GitHub Pages URL loads successfully
- Page renders without layout breakage
- All required sections exist in the DOM
- No console errors on load

## Verification Checklist
- [ ] URL loads on desktop
- [ ] URL loads on mobile
- [ ] Pico.css styles applied
- [ ] All sections present
- [ ] No console errors

## Completion Evidence
- Live GitHub Pages URL
- Commit hash where site scaffold was added
